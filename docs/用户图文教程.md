# HFE 角色定制工具图文教程

适用对象：第一次使用 `HFE角色定制工具.exe` 的玩家。

本教程里的图片暂时使用占位说明。正式发布前，可以按每个“建议插图”位置补上截图。

## 1. 下载与解压

拿到发布者提供的压缩包后，先把整个压缩包解压到一个普通文件夹里，例如：

```text
HFE角色定制工具\
```

解压后，文件夹里至少应该有这些文件：

```text
HFE角色定制工具.exe
使用说明.txt
图文教程.md
runtime/
```

不要直接在压缩包内部双击运行工具。一定要先解压。

`runtime` 文件夹里带着工具需要的 Java/JDK 环境。电脑没有安装 Java 也可以使用，但不要删除这个文件夹。

> 建议插图 01：解压后的文件夹截图，重点框出 `HFE角色定制工具.exe`、`runtime` 文件夹和教程文件。

## 2. 启动工具

双击 `HFE角色定制工具.exe`。

工具会直接进入主界面。

如果双击后没有反应，先确认：

- 已经把压缩包完整解压出来，不是在压缩包内部运行。
- `runtime` 文件夹还在工具同一目录。
- 杀毒软件没有拦截 `HFE角色定制工具.exe`。

> 建议插图 02：双击工具后进入主界面的截图。

## 3. 认识主界面

进入工具后，左侧是项目列表和常用按钮，右侧是当前项目的编辑区域。

常用按钮：

- `刷新项目`：重新读取当前工具目录下的项目。
- `新建角色`：创建一个新的自定义角色项目。
- `打开项目`：打开已有角色项目文件夹。

右侧有三个页面：

- `编辑角色`：填写角色 ID、名称、目标游戏版本等基础信息。
- `贴图与技能`：替换角色贴图，设置指定动作帧生成道具。
- `校验导出`：检查项目并导出可运行的游戏 EXE。

> 建议插图 03：工具主界面截图，标注左侧项目列表、`新建角色` 按钮、三个页面标签。

## 4. 新建角色项目

点击左侧 `新建角色`。

新建窗口里，操作顺序是：

1. 先选择 `目标游戏 EXE/SWF`。
2. 等工具读取目标版本。
3. 再选择该版本可用的 `角色模板`。
4. 填写 `项目名称`。
5. 填写 `角色 ID`。
6. 点击 `创建`。

这里的顺序很重要：不同 HFE 版本拥有的角色和道具不完全一样，所以必须先选游戏版本，再选角色模板。

> 建议插图 04：新建角色窗口截图，重点框出 `目标游戏 EXE/SWF`、`选择目标`、`角色模板`、`项目名称`、`角色 ID`、`创建`。

### 角色 ID 怎么填

推荐使用英文小写、数字和下划线，例如：

```text
ray0
app0
cx0
z_wrestle0
```

不要使用中文、空格或特殊符号。

部分角色模板对 ID 长度更敏感。比如某些模板原本是 4 个字符，导出时也更适合使用 4 个字符的 ID。工具会在校验时给出提示，按提示修改即可。

## 5. 编辑角色基础信息

项目创建后，会进入 `编辑角色` 页面。

这里可以修改：

- `项目名称`
- `角色 ID（按此 ID 导出）`
- `角色名称`
- `中文名称`
- `角色说明`
- `目标游戏 EXE/SWF`
- `目标类型`

改完后，点击 `保存并校验`。

如果只是普通玩家使用，通常只需要改角色名称和说明；目标游戏版本一般在新建时已经选好了。

> 建议插图 05：`编辑角色` 页面截图，重点框出角色 ID、角色名称、目标游戏路径、`保存并校验` 按钮。

## 6. 替换角色贴图

切到 `贴图与技能` 页面，找到 `部位贴图` 区域。

每个部位都可以选择：

- 保持模板原样。
- 使用新的图片文件。

建议只替换你确认对应的部位，例如头部、身体、手臂等。不同角色的骨骼和贴图命名可能不完全一样，如果替换后游戏内错位，可以回到工具里改回模板原样，或换一张更匹配的图。

> 建议插图 06：`贴图与技能` 页面里 `部位贴图` 区域截图，标出某个部位的贴图选择控件。

### 贴图建议

- 图片尽量使用 PNG。
- 图片尺寸尽量接近原角色对应部位。
- 如果只是想先测试导出流程，可以全部保持模板原样。
- 如果导出游戏能进，但外观错位，通常是贴图尺寸或部位不匹配，不一定是工具导出失败。

## 7. 设置技能帧生成道具

在 `贴图与技能` 页面上方，可以设置某个动作的某一帧生成道具。

基本流程：

1. 在 `技能动作` 下拉框里选择动作。
2. 在 `动作内帧` 下拉框里选择帧。
3. 在道具槽里选择要生成的道具。
4. 设置 `x / y / z` 出现位置。
5. 设置 `vx / vy / vz` 初速度。
6. 保存项目。

每一帧最多可以设置多个道具槽。空槽表示不生成道具。

> 建议插图 07：`技能动作`、`动作内帧` 和道具槽区域截图，标出道具下拉框、位置参数、速度参数。

### 常用参数理解

- `x`：横向位置。数值越大，越靠前。
- `y`：纵向位置。通常负数会更靠上。
- `z`：高度或层次相关位置。
- `vx`：横向速度。正数通常向前飞。
- `vy`：纵向速度。
- `vz`：高度方向速度。

如果只是想让道具正常出现在角色前方，可以先使用默认值，再进游戏测试效果。

## 8. 校验并导出游戏

切到 `校验导出` 页面。

点击：

```text
执行完整校验并导出
```

工具会先检查项目配置，再生成新的游戏 EXE。

导出成功后，结果区域会显示导出的 EXE 路径。通常导出的游戏会出现在工具所在目录。

> 建议插图 08：`校验导出` 页面截图，重点框出 `执行完整校验并导出` 按钮和导出结果区域。

## 9. 测试导出的游戏

导出后，双击新生成的游戏 EXE。

建议按下面顺序检查：

1. 游戏能正常打开。
2. 加载时没有警告弹窗。
3. 不会卡在加载界面。
4. 角色列表或 VS 选人界面能看到新角色。
5. 能选择新角色进入战斗。
6. 移动、跳跃、普通攻击正常。
7. 设置过的技能帧能正常生成道具。
8. 道具攻击判定正常。
9. 如果使用登录版本，结算时没有账号异常、封号或 hack 提示。

## 10. 常见问题

### 需要自己安装 Java 吗

免 Java 发布版不需要自己安装 Java。压缩包里的 `runtime` 文件夹已经带了工具需要的 JDK。不要删除 `runtime` 文件夹，否则导出和目标版本读取可能失败。

### 选择目标版本后，角色模板变了

这是正常的。工具会按当前目标游戏版本读取角色模板。某些角色不是每个版本都有。

### 换了目标版本后，道具列表变了

这是正常的。每个版本的 `itemSpt` 可能不同，工具会按当前目标版本刷新道具目录。

### 导出失败

先看 `校验导出` 页面里的错误提示。常见原因包括：

- 角色 ID 不符合要求。
- 目标游戏文件路径无效。
- 选择了当前版本不可用的角色或道具。
- 某些贴图或素材文件不存在。

按提示修正后，再重新导出。

### 导出的游戏加载时警告或卡住

不要继续发布这个 EXE。把导出日志和游戏表现反馈给发布者或工具作者。

导出日志通常和导出的 EXE 在同一目录，文件名里会带：

```text
_export_log.md
```

### 贴图没有变化

确认：

- 对应部位不是 `保持模板原样`。
- 选择的图片文件没有被移动或删除。
- 导出后打开的是最新生成的 EXE，而不是旧 EXE。

### 贴图错位

可能是图片尺寸、部位或模板体型不匹配。可以换更接近原角色部位的图片，或先改回模板原样。
